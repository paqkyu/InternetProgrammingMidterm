Task Allocation System

How tasks are created, stored, managed and displayed.

The StefGym is a website that allows users to create and manage their tasks.

Each task must contain a name,description, due date, and a priority level. After being added a task has a status of completed or pending. Users can also edit, delete, filter, complete, and sort tasks. Tasks are stored in localStorage so it remains available even after the page is refreshed.

Task Creation

Users can enter a task name, description, due date, and priority using the task form. Task name and decription have character constraints, these are also backed by a javascript function so that DOM manipulation cannot be done.

Data Persistence

localStorage saves the tasks array as a JSON and restores it whenever the page is opened.

Coding Decisions

Documentation of the tasks page, and About page.

Tasks Page

Jquery was selected for form event handling, reading input values, generating table rows, and updating the task summary dynamically.

The task data is held in a javascript array and saved using localStorage.

Filtering and sorting were implemented using temporary copies of the task array, allowing users to sort tasks without changing the originally stored data.

  • jQuery form and button event handling
  • LocalStorage and task persistence
  • Task creation, editing, deletion, and completion
  • Status and priority filtering
  • Task analytics
  • HTML and javascript validation

About page

The about page was created to showcase the "team members". Boostrap cards were used to display the staff photographs, names, and roles.

The boostrap cards allow the staff cards to adapt to all devices and their screen resolutions. A bootstrap carousel is also implemented to introduce staff.

jquery adds interactive effects to the staff cards when the user hovers over them.

  • Responsive Grid
  • Staff cards
  • Image carousel
  • jQuery hover events

Personal reflections

Challenges and how they were addressed.

One of the main challenges was trying to create a responsive layout. Bootstrap was used to create the responsive layout, and media queries were used to cater to smaller devices

Task Persistence
